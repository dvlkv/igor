# Igor

<img width="256" height="256" align="left"  alt="Igor&#39;s mad scientist experiment" src="https://github.com/user-attachments/assets/64b55ae0-5b95-44ac-a20a-bc3535625d14" />

Igor is an autonomous coding agent that acts as your teammate — not a piece of software, but a curated set of tools, prompts, and workflows composed together to form one.

Rather than building a standalone application, Igor takes a different approach: it leverages existing AI-powered coding environments and augments them with carefully crafted prompts, agent skills, MCP integrations, and conventions that turn a general-purpose LLM into a reliable, opinionated engineering partner.  
<br/><br />



## What Igor is

- **A toolkit, not an app.** Igor is a collection of configurations, prompt engineering, agent skills, and tool integrations designed to be dropped into your development workflow.
- **A teammate, not an assistant.** Igor doesn't just answer questions — it plans, executes, debugs, and iterates on code autonomously, much like a junior developer on your team.
- **Composable by design.** Every piece — prompts, skills, MCP servers, rules — can be swapped, extended, or replaced independently.

## How it works

Igor is assembled from:

- **Prompts & rules** that shape agent behavior, coding style, and decision-making
- **Agent skills** that encode domain-specific knowledge and workflows
- **MCP servers** that connect the agent to external services (Linear, PostHog, documentation sources, etc.)
- **Conventions** (commit style, task management, mode switching) that keep the agent's output consistent and predictable

The result is an agent that understands your project context, follows your team's conventions, and can take on multi-step engineering tasks end-to-end.
